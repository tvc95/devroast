# Specs — Formato

Criar spec em `specs/<feature-name>.md` antes de implementar.

## Estrutura

```markdown
# Especificação: <Nome da Feature>

## 1. Visão Geral
Breve descrição do que é e para que serve.

---

## 2. Referência
Link para referência externa (opcional).

---

## 3. Análise de Alternativas
Tabela comparativa de ferramentas/approaches (prós vs contras).

---

## 4. Arquitetura Proposta
- Stack/Tecnologias
- Componentes (estrutura de arquivos)
- Fluxo de dados

---

## 5. Funcionalidades
### Obrigatórias
- [ ] ...

### Opcionais (Futuro)
- [ ] ...

---

## 6. Questões em Aberto
### ❓ Pergunta
**Resposta**: ...

---

## 7. To-Dos para Implementação
- [ ] **TASK-1**: ...
```

## Regras

- Cada spec = 1 feature
- Usar Markdown padrão
- Incluir sempre To-Dos para guiar implementação
- Questões em aberto devem ser respondidas antes de começar a codar
